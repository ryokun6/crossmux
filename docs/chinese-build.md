# Chinese Build (`gh_release_cn`)

Comprehensive guide to the Simplified-Chinese variant of CrossPoint Reader:
what's built-in, how it's wired, memory/flash budget, and how to extend.

> This is a **separate firmware image** gated by `ENABLE_CHINESE_VERSION`.
> The default international firmware (`gh_release`) is byte-for-byte unchanged
> by anything in this document.

---

## TL;DR

- `pio run -e gh_release_cn` builds a CN-only firmware: English + 简体中文 UI,
  ~3,500 common Chinese characters baked into the firmware as 6 font sizes
  (8/10/12/14/16/18 pt Regular), NotoSerif/OpenDyslexic/Ubuntu dropped.
- `.pio/build/gh_release_cn/firmware.bin` ≈ 5.1 MB / 6.25 MB partition (78%).
- Runtime ≈ 63 KB persistent heap on top of baseline (32 KB hot-group reserve +
  ~31 KB UI prewarm cache for UI_10 + UI_12 over the hot-UI CJK subset).
- International build is untouched (byte-for-byte verified).

---

## Architecture decisions

### 1. Why a separate firmware variant, not a runtime toggle?

A full CJK glyph set won't fit alongside the 22 international languages and
NotoSerif/NotoSans/OpenDyslexic/Ubuntu within a single 6.25 MB OTA partition.
The CN build drops all of that to free flash for ~1.7 MB of NotoSansSC
bitmaps. International users get exactly the firmware they had before.

The gate is the pre-existing `ENABLE_CHINESE_VERSION` macro (originally used
only for the Chinese-Chess game). All CN-specific code is `#ifdef`-guarded
behind it so the international build sees zero impact.

### 2. Font strategy

Single typeface (Noto Sans SC, Regular only) at 6 sizes matching the Latin
size lineup:

| Slot | Latin original | CN |
|---|---|---|
| `smallFont` (status bar) | NotoSans 8pt | NotoSansSC 8pt |
| `UI_10` (status bar accents) | Ubuntu 10pt | NotoSansSC 10pt |
| `UI_12` (menus, list rows) | Ubuntu 12pt | NotoSansSC 12pt |
| reader 12pt | NotoSerif/NotoSans/OpenDyslexic 12pt | NotoSansSC 12pt |
| reader 14pt | …14pt | NotoSansSC 14pt |
| reader 16pt | …16pt | NotoSansSC 16pt |
| reader 18pt | …18pt | NotoSansSC 18pt |

**No bold / italic / bold-italic** for CN. `EpdFontFamily`'s nullptr-fallback
collapses those style requests back to Regular. Italic markup in EPUBs renders
upright; bold renders as Regular. Acceptable for a CN-only typeface (CJK
italic is non-idiomatic, and bold CJK at small e-ink sizes is muddy anyway).

NotoSans/NotoSerif/OpenDyslexic font-family selectors are hidden in the CN
font picker (only "Noto Sans" is shown), but the underlying `NOTOSERIF_*_FONT_ID`
constants stay registered — they alias to the same NotoSansSC data so any
legacy callsite (e.g. `SudokuGameActivity` hard-coded `NOTOSERIF_14_FONT_ID`)
keeps working without code changes.

### 3. Character set

3,500 Hanzi from **通用规范汉字表 一级常用字** (PRC State Council, 2013), plus
CJK punctuation, fullwidth ASCII, and any extra CJK char actually present in
the simplified_chinese.yaml translations:

- Source list: `lib/EpdFont/scripts/charsets/zh_cn_3500.txt` (vendored from
  [elephantnose/characters](https://github.com/elephantnose/characters))
- Build script: `lib/EpdFont/scripts/charsets/build_zh_cn_charset.py`
  produces `zh_cn_common.txt` (~3,665 codepoints) by union of:
  1. the 3,500-char base
  2. every CJK codepoint in `lib/I18n/translations/simplified_chinese.yaml`
  3. CJK punctuation block (U+3000–U+303F)
  4. Fullwidth ASCII (U+FF01–U+FF5E)
  5. A handful of typography marks (·, —, …, ★, ☆, 「」『』)

Coverage trade-off: ~99% of modern Chinese reading content. Low-frequency
chars (e.g. 鬓 U+9B13) render as ▯ (U+FFFD replacement). Documented per the
user's "去除生僻低频用字" decision.

### 4. Group decompression budget

Glyph data ships DEFLATE-compressed in "script groups" generated by
`fontconvert.py`. `GROUP_MAX_UNCOMPRESSED_BYTES` caps each group at **32 KB**
(was 64 KB; halved because CN groups are denser and 64 KB transient mallocs
on a fragmented heap caused cold-boot crashes).

The decompressor (`lib/EpdFont/FontDecompressor.cpp`) pre-reserves a 32 KB
hot-group buffer at `init()` (`FontDecompressor.h: MAX_GROUP_SIZE`). All
subsequent `hotGroup.resize(N ≤ 32K)` calls reuse that capacity — no runtime
malloc, no `bad_alloc` → `terminate` crash even on a fragmented heap.

If you change `GROUP_MAX_UNCOMPRESSED_BYTES` in fontconvert.py, update
`MAX_GROUP_SIZE` in FontDecompressor.h to match. The comment in both files
references the constraint.

### 5. UI font prewarm

To eliminate per-frame decompression stutter on lists/menus, `prewarmUIFonts()`
(`src/main.cpp`) pre-decompresses the hot-UI CJK codepoints for the current
language into `FontDecompressor::pageSlots[]`:

- **Scope**: `UI_10_FONT_ID` + `UI_12_FONT_ID`. UI_12 covers menu titles,
  list-row main text, and dialog body. UI_10 covers LyraTheme's list rows
  and tab labels — adding it removed the per-page "first new char" stutter
  when navigating Settings/Font/Language. Reader fonts use the existing
  per-page prewarm path (`FontCacheManager::PrewarmScope`). `smallFont`
  (status bar, button hints, list subtitles) is intentionally not prewarmed
  — its cache misses are concentrated on rare characters and prewarming the
  full subset would push ~7 KB heap for limited gain. Revisit if `logStats`
  shows it's a hotspot.
- **Charset**: `CORE_CHARACTER_SETS[currentLang]` (generated by
  `gen_i18n.py`), filtered to codepoints ≥ 0x80. This is the **hot-UI
  subset** — the union of CJK chars referenced by STR_\* keys used in
  `HOT_PREWARM_DIRS` (`src/main.cpp`, `src/activities/{boot_sleep,home,util,
  components}/`, plus the Settings/Language/Font/StatusBar/ButtonRemap/
  ClearCache activities). It is **not** the full per-language charset:
  for ZH_CN it shrinks 358 chars → ~240 (~33% reduction), saving ~31 KB
  pageBuf vs the full set. Cold UI (OTA progress, KOReader/OPDS dialogs,
  Calibre instructions, network errors) falls through to the hot-group
  path on first render — ~10 ms stutter per missing char, then session-
  cached. `CHARACTER_SETS[]` (full union) is still emitted alongside and
  used by EPUB Reader's `PrewarmScope`. ASCII chars live in the Latin
  hot-group; they decompress once on first render and stay cached.
- **Cost**: ~31 KB persistent heap total (~13 KB for UI_10, ~18 KB for
  UI_12; ~240 chars per size × bitmap + per-glyph entry table). Each size
  occupies one of `MAX_PAGE_SLOTS = 4` slots; currently 2/4 used.
- **Trigger**: at boot in `setup()` after `setupDisplayAndFonts()`, and on
  language change in `LanguageSelectActivity::handleSelection()`.
- **Instrumentation**: `FontCacheManager::logStats("prewarm-done")` runs at
  the end of `prewarmUIFonts()`, and `logStats("settings-exit")` runs on
  `SettingsActivity::onExit` — both visible with LOG_LEVEL=2. Use the
  hits/misses + avg getBitmap µs to decide whether to prewarm `smallFont`
  too.

#### 5.1 prewarmCache reuses hotGroup as temp buffer

`FontDecompressor::prewarmCache()` decompresses each needed group into the
already-reserved `hotGroup` vector (`init()` pre-allocates 32 KB while heap
is contiguous), instead of `malloc`-ing a fresh 32 KB temp block per group.
The old per-group malloc path failed for 16/19 UI_12 groups on a fragmented
heap (~19 KB MaxAlloc observed mid-boot), silently leaving most CJK glyphs
unprewarmed and producing 20 ms/char `getBitmap` stutter on the first
Settings open. With the reuse, prewarm reports `0 missed` deterministically.

#### 5.2 Extending or relocating `HOT_PREWARM_DIRS`

The hot-UI scan list lives in `scripts/gen_i18n.py` near the top, alongside
`HOT_PREWARM_SKIP_PREFIXES` (e.g. `STR_OTA_`, `STR_KOREADER_`, `STR_CALIBRE_`).
If you add a new Activity that contains frequently-shown CN text, either:

1. **Add the file or directory to `HOT_PREWARM_DIRS`** (preferred when the
   whole module is hot), then rerun `python scripts/gen_i18n.py …`. The CN
   build will pick it up on the next `pio run -e gh_release_cn`.
2. **Reuse an existing STR_\* key** that is already prewarmed (zero cost).

If a hot-path Activity sneaks in via a directory that is in `HOT_PREWARM_DIRS`
but uses cold STR\_\* prefixes (e.g. `STR_KOREADER_*` mixed in a Settings
activity), accept that those chars render once via the hot-group fallback —
unless they're a stutter hotspot, in which case loosen `HOT_PREWARM_SKIP_PREFIXES`.

The full `CHARACTER_SETS[]` union still exists (so EPUB content and other
callers stay covered), and the `CORE_CHARACTER_SETS[]` declaration is itself
gated with `#ifdef ENABLE_CHINESE_VERSION` to keep the international firmware
byte-stable.

### 6. i18n filtering

CN builds embed **English + Simplified Chinese only** (not all 22 international
language YAMLs). `scripts/gen_i18n.py` auto-detects `ENABLE_CHINESE_VERSION`
in `BUILD_FLAGS`/`CPPDEFINES` from SCons env and applies a hard-coded
`["english", "simplified_chinese"]` whitelist. The international build
correspondingly excludes `simplified_chinese.yaml` so its size stays byte-
stable.

The same `--whitelist` flag is also passed explicitly by the simulator's
CMake build (see [Simulator integration](#simulator-integration)).

### 7. Apps deferred

Game/Avatar UI strings (`STR_SUDOKU_*`, `STR_GOMOKU_*`, `STR_CHINESE_CHESS_*`,
`STR_GAME_*`, `STR_UGLY_AVATAR`, `STR_SAVE`, `STR_RANDOM`) are intentionally
**absent from `simplified_chinese.yaml`** — they fall back to English via
gen_i18n.py's existing fallback mechanism. See
`lib/I18n/translations/simplified_chinese.TODO.md` for the deferred-key list.

## Memory budget

### Flash (per OTA partition: 6.25 MB)

| Component | Bytes |
|---|---|
| CrossPoint code + libraries | ~2.0 MB |
| Chinese Chess font (`chinese_chess_16`) | ~10 KB |
| NotoSansSC 8pt | ~190 KB binary |
| NotoSansSC 10pt | ~230 KB binary |
| NotoSansSC 12pt | ~300 KB binary |
| NotoSansSC 14pt | ~340 KB binary |
| NotoSansSC 16pt | ~400 KB binary |
| NotoSansSC 18pt | ~480 KB binary |
| i18n strings (EN + ZH_CN) | ~15 KB |
| Misc (icons, HTML, certs, etc.) | ~1.1 MB |
| **Total** | **~5.1 MB (78% of 6.25 MB)** |

### Runtime heap (CN-specific, persistent)

| Source | Bytes |
|---|---|
| `FontDecompressor::hotGroup` reserve | 32 KB |
| `FontDecompressor::hotGlyphBuf` reserve | 1 KB |
| UI_10 prewarm pageSlot (~240 chars × ~96B + entries) | ~13 KB |
| UI_12 prewarm pageSlot (~240 chars × ~140B + entries) | ~18 KB |
| **Total CN-specific persistent** | **~64 KB** |

Free heap after boot+prewarm on hardware: ~62 KB (largest contiguous block
~55 KB on a representative cold boot). Was ~28 KB / 19 KB MaxAlloc before
the CORE-subset prewarm and the hotGroup-reuse change in §5.1.

Activity transitions and book-cover loading temporarily consume an
additional 40-50 KB on top of the persistent baseline (Home Activity is
the heaviest consumer); the larger contiguous free block after the §5/§5.1
changes is why Home now renders covers in ~230 ms instead of the
~1000 ms seen pre-optimisation, and why Settings sees MaxAlloc ~40 KB
instead of the previous ~10 KB.

## Build & regenerate

### Standard build (commits font headers already in tree)

```sh
pio run -e gh_release_cn
```

Generates `.pio/build/gh_release_cn/firmware.bin`. No font regen, no network.

### Regenerating CN font headers

Required when changing:
- The charset (`zh_cn_3500.txt` or `simplified_chinese.yaml`)
- The size list (currently 8/10/12/14/16/18)
- `GROUP_MAX_UNCOMPRESSED_BYTES` in `fontconvert.py`
- Source font version (NotoSansSC)

```sh
# 1. Ensure source font is present (downloads if missing)
bash lib/EpdFont/builtinFonts/source/NotoSansSC/fetch.sh

# 2. Rebuild charset (only needed if yaml or zh_cn_3500.txt changed)
python lib/EpdFont/scripts/charsets/build_zh_cn_charset.py

# 3. Regen all 6 CN headers
cd lib/EpdFont/scripts
BUILD_CN_FONTS=1 bash convert-builtin-fonts.sh
cd ../../..

# 4. Rebuild firmware
pio run -e gh_release_cn
```

Generation takes ~10 s for all 6 sizes (NotoSansSC is dense and fontconvert
does kerning/ligature extraction once per size).

### Required dev dependency

`fontconvert.py` needs `freetype-py` and `fonttools` — install into a venv:

```sh
python3 -m venv .venv-fonts
.venv-fonts/bin/pip install -r lib/EpdFont/scripts/requirements.txt
```

(`.venv-fonts/` is gitignored.)

## Simulator integration

`./simulator.sh` builds the desktop simulator with `ENABLE_CHINESE_VERSION=1`
(matches `gh_release_cn`). Key bits in `simulator/CMakeLists.txt`:

- `add_custom_target(i18n_generated ALL ...)` — re-runs `gen_i18n.py` on
  **every** `cmake --build` (not just when YAML mtime changes), defeating the
  pio/simulator i18n-file race.
- Passes `--whitelist english,simplified_chinese` so the simulator's i18n
  blob matches `gh_release_cn` exactly.

If you run `pio run -e gh_release` (international) between simulator runs,
the simulator's next build will overwrite the pio-generated 22-language
`I18nKeys.h` with the 2-language CN version. This is expected — pio rebuilds
restore it.

To run the simulator after a fresh checkout:

```sh
git submodule update --init   # open-x4-sdk needed for shared libs
./simulator.sh                 # installs SDL/cmake deps, builds, launches
```

## Extending the CN build

### Add bold support

Memory cost: rough doubling of font flash (each size × 2 weights ≈ +1.7 MB
binary). Likely won't fit; requires dropping a reader size.

1. Generate `notosanssc_{8,10,12,14,16,18}_bold.h` via fontconvert
2. Add `EpdFont notosansscNNBoldFont(&notosanssc_NN_bold)` declarations in
   the CN branch of `src/main.cpp`
3. Pass them to `EpdFontFamily` constructors as the second arg
4. Re-check `pio run -e gh_release_cn` size; may need to drop 18pt or shrink
   the charset

### Add italic / bold-italic

CJK italic is unidiomatic and visually awkward at small e-ink sizes. Don't.

### Prewarm more UI sizes

`UI_10` + `UI_12` are already prewarmed (see §5). To add `smallFont` (status
bar, button hints, list subtitles):

```cpp
// in prewarmUIFonts()
fontCacheManager.prewarmCache(SMALL_FONT_ID, nonAscii, 0x01);
```

Cost: +~7 KB persistent heap (8pt glyphs are smaller). Before doing it, check
the `settings-exit` line emitted by `FontCacheManager::logStats` after a
representative menu session — if cacheMisses is single-digits or
hotGroupBytes is consistently 0, the prewarm isn't worth it.

`MAX_PAGE_SLOTS = 4` total; we currently use 2 slots (UI_10 + UI_12). Each
additional font consumes one slot.

### Reduce flash by trimming sizes

If a future change pushes `gh_release_cn` past 6.25 MB:

1. **First fallback**: drop 18pt, point reader 18pt slot at 16pt data (visual
   regression at "Extra Large" font size but reader still works). Saves
   ~500 KB binary.
2. **Second fallback**: reduce charset from 通用规范一级 (3,500) to 现代汉语
   常用字一级 (2,500). Saves ~30%. Edit `build_zh_cn_charset.py` to swap the
   base source.

### Translate Apps strings

Add the deferred keys from `simplified_chinese.TODO.md` to
`simplified_chinese.yaml`. No build-system changes needed; the gen_i18n.py
pipeline will pick them up automatically.

## Known limitations

- **Glyph coverage**: chars outside `zh_cn_common.txt` render as ▯. Affects
  rare proper nouns and classical-Chinese text. EPUB content is the main
  surface where you'd notice this — UI strings are guaranteed covered
  because the build script unions in the YAML's CJK chars.
- **No serif option in CN build**: NotoSerif and OpenDyslexic font families
  are not shipped. Font picker shows only Noto Sans.
- **Bold/italic renders as Regular**: see Architecture §2.
- **Font picker enum stability**: `CrossPointSettings::FONT_FAMILY` keeps
  values `NOTOSERIF=0, NOTOSANS=1, OPENDYSLEXIC=2` even in CN (no enum
  shrinking) to keep saved settings forward-compatible. `getReaderFontId()`
  maps any of these to NotoSans IDs in the CN branch.
- **Cold-boot crash class**: previously OOM'd if HomeActivity tried to render
  Chinese book titles before any prewarm. Fixed by the hot-group reserve.
  If you change `MAX_GROUP_SIZE` ≠ `GROUP_MAX_UNCOMPRESSED_BYTES`, the
  crash class returns; comments in both files warn about it.
- **Cold-UI stutter on first display**: STR\_\* keys excluded from
  `HOT_PREWARM_DIRS` (OTA progress, KOReader/OPDS dialogs, Calibre
  instructions, network errors) render their first glyph via the hot-group
  fallback — ~10 ms per missing CJK char on first display, then cached for
  the rest of the session. Acceptable for the UI surfaces it covers.
  Adding a hot path here later means adding the file/directory to
  `HOT_PREWARM_DIRS` in `scripts/gen_i18n.py` (see §5.2).
